# 👋 OwlRoadmap Community Guidelines

Welcome to the OwlRoadmap community! We're a developer-first, open-source project and we want every contributor — regardless of background or experience level — to feel safe, respected, and empowered.

Please read these guidelines before participating in any community channel (GitHub Issues, Pull Requests, or Discussions).

---

## 🌍 Our Values

| Value | What it means |
| --- | --- |
| **Openness** | We welcome contributors of all skill levels and backgrounds. |
| **Respect** | We treat every person with courtesy — even when we disagree. |
| **Collaboration** | We work together. No gatekeeping, no lone-wolf heroics. |
| **Beginner-friendliness** | We actively maintain `good first issue` tickets and offer mentoring. |
| **Quality** | We hold ourselves to high engineering and documentation standards. |

---

## 📣 Where to Participate

| Channel | Purpose |
| --- | --- |
| [GitHub Issues](https://github.com/SunilKumarKV/owlroadmap/issues) | Bug reports and feature requests |
| [GitHub Discussions](https://github.com/SunilKumarKV/owlroadmap/discussions) | Q&A, ideas, show-and-tell, polls |
| [Pull Requests](https://github.com/SunilKumarKV/owlroadmap/pulls) | Code contributions, docs improvements |

> ⚠️ Please **do not** open issues to ask general questions. Use **Discussions** instead.

---

## ✅ Do's

- **Search before opening** — Check if an issue or discussion already exists for your topic.
- **Be specific** — Provide reproduction steps, environment details, and expected vs. actual behaviour.
- **Be constructive** — Feedback should be actionable and kind. "This is broken" is not helpful; "This breaks when I click X because Y" is.
- **Follow conventions** — Use the correct branch prefix, commit type, and issue template.
- **Celebrate contributions** — React to PRs, thank people for helping, and encourage newcomers.

## ❌ Don'ts

- **Don't be rude, dismissive, or condescending** — Ever. To anyone.
- **Don't spam** — Avoid opening duplicate issues or bumping threads repeatedly.
- **Don't demand** — Maintainers are volunteers. Patience is appreciated.
- **Don't submit large PRs without discussing them first** — Open an issue or Discussion to align on approach.
- **Don't share others' private information** — Respect privacy in all forms.

---

## 🧑‍🏫 New to Open Source?

Welcome! Here are some great starting points:

1. Read [CONTRIBUTING.md](../CONTRIBUTING.md) for the full setup guide.
2. Browse [`good first issue`](https://github.com/SunilKumarKV/owlroadmap/labels/good%20first%20issue) labelled tickets.
3. Join a [Discussion](https://github.com/SunilKumarKV/owlroadmap/discussions) to introduce yourself.
4. Open a **Draft PR** early — you don't need to finish before asking for feedback.

---

## 🤝 Mentorship

First-time contributors are encouraged to request mentorship. Add the comment:

```
/mentor
```

...on any issue labelled `good first issue` and a maintainer will guide you through the implementation.

---

## ⚖️ Enforcement

Violations of these guidelines or the [Code of Conduct](../CODE_OF_CONDUCT.md) may result in a warning, temporary ban, or permanent ban from community channels depending on severity.

Report violations to **support@owlroadmap.com** — all reports are kept confidential.
