# Security Policy

## Supported Versions

We actively support and patch the latest minor release of the current major version.

| Version | Supported |
| ------- | --------- |
| v1.1.x  | ✅ Yes    |
| < v1.1  | ❌ No     |

## Reporting a Vulnerability

We take the security of OwlREADME seriously. If you find a security vulnerability, please do not report it publicly via GitHub issues. Instead, report it directly to our security response team:

* **Email**: support@owlreadme.com
* **Encryption**: If required, please coordinate with us via email to exchange PGP keys or set up a secure channel.

Please include the following details in your report:
* A detailed description of the vulnerability and its potential impact.
* Step-by-step instructions to reproduce the issue (including proof-of-concept scripts or payloads where possible).
* Any details about the environment or browser configurations used.

## Security Handling Process

Once a report is received, our team will:
1. Acknowledge receipt of the report within **48 hours**.
2. Investigate the issue and confirm the vulnerability.
3. Work on a fix and coordinate a release window.
4. Publish a security advisory and credit the reporter (unless anonymity is requested).

We ask that you follow **Responsible Disclosure** principles, giving us at least **90 days** to patch the issue before making any public disclosures.
